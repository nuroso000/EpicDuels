# Contributing Guidelines for EpicDuels

Thank you for your interest in contributing to this project. To ensure a smooth process, please follow these guidelines.

## Bug Reports
If you encounter a bug, please open an issue in the repository. Provide a detailed description of the problem, including:
- Steps to reproduce the issue.
- Expected behavior vs. actual behavior.
- Version information (Minecraft/Paper version and Java version).

## Feature Requests
We welcome suggestions for new features. Please open an issue and describe the proposed functionality and its benefits to the plugin.

## Pull Requests
To contribute code changes, please follow these steps:
1. Fork the repository.
2. Create a feature branch (`git checkout -b feature/your-feature-name`).
3. Commit your changes with descriptive messages.
4. Push to your branch and open a Pull Request.

## Technical Standards
- Compatibility: All code must be compatible with Java 21 and the Paper 1.21.1+ API.
- Code Quality: Ensure your code is readable and adheres to standard Java naming conventions.
- Documentation: Update the README or include comments for significant logic changes.
